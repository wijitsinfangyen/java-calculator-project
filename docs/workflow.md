# ขั้นตอนการทำงานของกลุ่ม (GitHub Workflow)

## โครงสร้าง Branch
- main : โค้ดที่เสร็จสมบูรณ์ พร้อมใช้งาน
- develop : รวมงานจากสมาชิกทุกคน
- feature/* : งานของสมาชิกแต่ละคน

## ขั้นตอนการทำงาน
1. Clone repository จาก GitHub
2. เปลี่ยนไปที่ branch develop
3. สร้าง feature branch ของตนเอง
4. เขียนโค้ดและ commit
5. push ขึ้น GitHub
6. สร้าง Pull Request เข้า develop
7. หัวหน้ากลุ่มตรวจสอบและ merge โค้ด

## หน้าที่หัวหน้ากลุ่ม
- สร้างและดูแล repository
- ตรวจ Pull Request
- แก้ merge conflict
- รวมโค้ดเข้า branch develop และ main
